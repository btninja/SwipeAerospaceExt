# About

**SwipeAeroSpaceExtended** is a fork of [MediosZ/SwipeAeroSpace](https://github.com/MediosZ/SwipeAeroSpace) that adds extra 3-finger gestures for [AeroSpace](https://github.com/nikitabobko/AeroSpace). It runs alongside the upstream app (different bundle ID), so you can keep both installed while migrating.

The base behavior — 3-finger swipe LEFT/RIGHT to change AeroSpace workspaces — comes from upstream. This fork adds 3F UP/DOWN, 3F tap, and fn-modified 3F swipes for window movement, monitor focus, and close.

# Gesture set

All 9 gestures are 3-finger. fn-modified variants require a hardware fn key (or a Karabiner/Vial firmware mapping that produces a real fn modifier).

| Gesture            | AeroSpace command                                  | Default toast        |
| ------------------ | -------------------------------------------------- | -------------------- |
| 3F swipe LEFT      | `workspace --wrap-around prev`                     | `Workspace ←`        |
| 3F swipe RIGHT     | `workspace --wrap-around next`                     | `Workspace →`        |
| 3F swipe UP        | `focus-monitor next`                               | `Monitor →`          |
| 3F swipe DOWN      | `workspace-back-and-forth`                         | `⇄`                  |
| 3F tap             | `layout floating tiling`                           | `Float toggle`       |
| fn + 3F LEFT       | `move-node-to-workspace --wrap-around prev`        | `fn ← Move prev`     |
| fn + 3F RIGHT      | `move-node-to-workspace --wrap-around next`        | `fn → Move next`     |
| fn + 3F UP         | `move-node-to-monitor next`                        | `fn ↑ Move to monitor` |
| fn + 3F DOWN       | `close`                                            | `fn ↓ Close`         |

3F swipe UP can be reverted to the upstream workspace overview HUD by toggling **Show Workspace Overview** under **Vertical Swipe** in Settings. The overview code is preserved; this fork just disables the trigger by default so the slot is free for `focus-monitor next`.

Each gesture pops a small bottom-center toast for ~0.8s. Toasts are anchored on the screen under the cursor.

# Settings

Available under the menu bar icon → Settings.

- **Horizontal Swipe** — sensitivity, finger count, natural direction, wrap, skip-empty, multi-workspace toggle, max steps per swipe.
- **Vertical Swipe** — toggle the workspace overview HUD on. When off (default in this fork), 3F up/down trigger `focus-monitor` and `workspace-back-and-forth`.
- **Tap** — enable 3-finger tap toggling between floating and tiling layouts.
- **Modifier Swipes** — enable fn-modified 3F swipes for window-movement ops.
- **General** — Launch at Login.

A one-time migration on first launch flips `swipeUpOverview` to false (gated by `migratedOverviewToggle` in UserDefaults) so users upgrading from upstream don't keep getting the overview on 3F up.

# Installation

Build from source until a Homebrew cask is published for the fork.

## Build from source

Install Xcode 16+ (the project uses the Xcode 16 project format, `objectVersion = 77`), then either:

- Open `SwipeAeroSpace.xcodeproj` and build the `SwipeAeroSpace` scheme. The output is `SwipeAeroSpaceExtended.app`.
- Or run `xcodebuild -project SwipeAeroSpace.xcodeproj -scheme SwipeAeroSpace -configuration Release -derivedDataPath build` from the project root.

The app needs access to global trackpad events. Grant `SwipeAeroSpaceExtended` access in **System Settings → Privacy & Security → Accessibility**.

## Homebrew cask (when published)

Once a cask is in your tap:

```bash
brew install --cask <owner>/tap/swipeaerospace-extended
```

The cask formula is generated by `scripts/build-brew-cask.py --build-version <version> --repo-slug <owner>/swipeaerospace-extended`.

## Nix flake (nix-darwin)

For a `darwin-rebuild` config, after publishing the cask:

```nix
homebrew = {
  taps = [
    "nikitabobko/tap"               # AeroSpace itself
    "<owner>/tap"                   # this fork
  ];
  casks = [
    "nikitabobko/tap/aerospace"
    "<owner>/tap/swipeaerospace-extended"
  ];
};
```

# Usage

After install, grant Accessibility permission and launch the app. Gestures fire immediately. Toggle individual behaviors in Settings.
